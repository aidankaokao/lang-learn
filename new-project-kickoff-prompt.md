# 新專案起手 prompt（複製貼到新 session 的第一則訊息）

> 用法：新專案已放好 `CLAUDE.md`（根目錄）、`reference/`（backend/frontend/deploy 慣例）、
> 並在 `reference/PROJECT-REQUIREMENTS.md` 填好需求後，開一個新的 Claude Code session，把下面這段貼進去。

---

我要開始一個新專案。請依序：

1. 讀專案根目錄的 `CLAUDE.md`（開發慣例入口）。
2. 讀 `reference/PROJECT-REQUIREMENTS.md`（我填好的本專案需求）。
3. 依需求，讀 `reference/` 內對應的慣例文件（前端風格 / backend / deploy 等），一律照裡面的慣例做，不要自行發明架構或風格。
4. 開工前，把 `CLAUDE.md` 的「開工前確認清單」中**需求說明沒寫清楚**的項目一次列給我確認；其餘照 reference 慣例當預設。
5. 確認後，先給我一個簡短的建置計畫（要建哪些檔、用哪些 reference 起手檔），再開始動手。

需求以 `reference/PROJECT-REQUIREMENTS.md` 為準；若有和 reference 慣例衝突之處，先問我。
